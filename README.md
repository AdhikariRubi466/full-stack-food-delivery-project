## 📖 About the Project

<table>
<tr>
<td>

**Tomato** is a production-grade, full-stack food delivery web application built with the modern JavaScript ecosystem. It provides a seamless end-to-end experience — from discovering mouth-watering dishes to placing a secure order — all wrapped in a beautifully crafted, responsive UI.

This project demonstrates a complete separation of concerns across:
- A **customer-facing storefront** for browsing, filtering, and ordering food
- A **full-featured admin dashboard** for managing inventory and processing orders
- A **RESTful backend API** built with Node.js and Express that powers all operations

> 🧑‍🍳 *"Delivering not just food — but an experience."*

</td>
</tr>
</table>

## 📁 Project Structure

```
📦 FULL-STACK-FOOD-DELIVERY/
│
├── 📂 backend/                        # Node.js + Express API Server
│   ├── 📂 data/                       # Static data layer
│   │   ├── 🟨 foodData.js             # Food catalog: 200+ items, categories, prices, ratings
│   │   └── 🟨 ordersData.js           # Orders seed data & schema
│   │
│   ├── 📂 routes/                     # Express route handlers
│   │   ├── 🟨 foodRoutes.js           # GET /food, POST /food, PUT /food/:id, DELETE /food/:id
│   │   └── 🟨 orderRoutes.js          # GET /orders, POST /orders, PATCH /orders/:id
│   │
│   ├── 📂 node_modules/               # Backend dependencies
│   ├── 🔒 .env                        # Environment variables (PORT, JWT_SECRET, etc.)
│   ├── 📋 package.json                # Backend package manifest
│   ├── 📋 package-lock.json           # Dependency lock file
│   └── 🟨 server.js                   # Express app entry point (CORS, routes, middleware)
│
├── 📂 frontend/                       # React.js + Vite Customer & Admin UI
│   ├── 📂 node_modules/               # Frontend dependencies
│   ├── 📂 public/                     # Static public assets
│   │   └── 🖼️  (images, icons, favicon)
│   │
│   ├── 📂 src/                        # Source code root
│   │   ├── 📂 components/             # Reusable UI components
│   │   │   ├── Navbar/                # Top navigation bar
│   │   │   ├── Footer/                # Site footer
│   │   │   ├── FoodCard/              # Individual food item card
│   │   │   ├── CategoryFilter/        # Pill filter tabs
│   │   │   └── Toast/                 # Notification toasts
│   │   │
│   │   ├── 📂 pages/                  # Page-level components
│   │   │   ├── Home/                  # Landing hero page
│   │   │   ├── Menu/                  # Full menu with filters & search
│   │   │   ├── Cart/                  # Cart & checkout flow
```

## 🔌 API Reference

### 🍽️ Food Endpoints

| Method | Endpoint | Description | Request Body |
|--------|----------|-------------|-------------|
| `GET` | `/api/food` | Get all food items | — |
| `GET` | `/api/food?category=Burger` | Filter by category | — |
| `GET` | `/api/food/:id` | Get single item by ID | — |
| `POST` | `/api/food` | Add new food item | `{ name, category, price, rating, description, image }` |
| `PUT` | `/api/food/:id` | Update a food item | `{ name?, category?, price?, rating?, description?, image? }` |
| `DELETE` | `/api/food/:id` | Delete a food item | — |
